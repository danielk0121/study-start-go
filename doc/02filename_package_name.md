func After(d Duration) <-chan Time { } 이거는 sleep.go 파일에 time 패키지로 되어있고,  
func (t Time) After(u Time) bool { } 이거는 time.go 파일에 time 패키지로 되어 있던데  

여러 파일에 걸쳐서 time 패키지 작성 했다고 보면 맞는건가 ?

### 1. 여러 파일, 하나의 패키지

Go에서는 **같은 디렉토리에** 있고 파일 상단에 `package time`이라고 적혀 있다면,   
파일 이름이 다르더라도 **모두 하나의 `time` 패키지**로 취급됩니다.  
- `time.go`: `Time 구조체` 정의와 시각 비교 메서드(`After`, `Before`) 등 핵심 로직
- `sleep.go`: `Sleep`, `After` (채널 리턴), `Tick` 등 타이머/런타임 대기 로직

컴파일러는 이 파일들을 모두 합쳐서 하나의 `time` 패키지로 빌드합니다.  
따라서 사용자 입장에서는 어느 파일에 들어있는지 상관없이 `time.After()` 라는 이름을 공유하게 됩니다.  

### 2. 왜 이렇게 나누어 놓았나요?

- 관리의 용이성: 코드가 수천 줄이 넘어가면 한 파일에서 보기 힘들기 때문에, **기능별로 파일을 쪼개** 관리합니다.  
- 런타임 의존성: 특히 `sleep.go` 에 있는 함수들은 
  - Go의 런타임 스케줄러 (고루틴을 재우고 깨우는 기능) 와 밀접하게 연결되어 있어, 
  - 순수 시간 계산 로직인 `time.go` 와 분리해 둔 것입니다.

### 3. 이름 충돌이 나지 않는 이유

Go에서는 **함수**와 **메서드**의 **호출 방식**이 완전히 다르기 때문에 이름이 같아도 충돌하지 않습니다.

1. **함수** 호출: `time.After(d)` (패키지 이름을 통해 호출)
2. **메서드** 호출: `t.After(u)` (특정 변수/인스턴스를 통해 호출)

결론적으로, **"Go의 패키지는 파일 단위가 아니라 디렉토리 단위다"** 라고 이해하시면 완벽합니다.  

혹시 Go에서 **함수명은 같지만 매개변수만 다른 '오버로딩'이 안 된다** 는 사실도 알고 계신가요?  
이 점이 자바와 가장 큰 차이 중 하나입니다.
